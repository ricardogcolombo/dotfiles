1.1.1. REGIONS

  • Each AWS Region is associated with a geographical name and a Region code.
  1. Availability zones
    • AWS cluster and data centers are around the world. when you have 1 or more datacenters connected through high speed low latency links in doing replication in case one is done this is an availability zone
    • When you have 2 availability zones connected throught high speed and low altency links doing replication this conforms a Region
  2. How to choose a region??
    • Compliance : Do you have a regulation requirement that your data needs to be in London? then you have to have your datacenters ina region (London)
    • Latency: chose the region close to your customers to give the resources faster.
    • Princing : One region could be more expencive than others.
    • Service availability: You want to make sure that the region that you are running has the services that you need.
  3. Global Edge Network
    • Consist in edge locations and regional edge caches.
    • you can use services like cloud front to cache content using edge locations.

1.3.3. AWS root user.

  • when you first create an aws account you begin twith a single sign in identity that has complete access to all aws service sand resources in the account. This identity is called AWS root user and is accessedy by signing with the email and pass that were used to create the account
  1. AWS root user credentials
    • the user has two sets of credentials associated with it.
    • One set of credentials is the email address an password that were used to create the account. This allows you to access the AWS Management Console. The second set of credentials is called access keys, which allow you to make programmatic request for AWS CLI or AWS API. (Access Key ID and Secret access key)
    • Delete your access keys to stay safe. If you dont have accesskey for your aws account root user dont create one unless you absolutely need to. if you have an access key for your aws account root user and want to delete
      1. in aws console, navigate to your user and upeper rigt section navigation bar > dropdown menu > My Security Credentials.
      2. Open the access keys section
      3. under Actions, choose DELETE
      4. Choose yes
  2. AWS root user best practices

    to ensure security

    • choose a strong password for the root user
    • enable multi factor
    • never share your user password or access key
    • disable or delete the access key associated with the root user
    • create an identity and access management IAM user for administrative tasks or everyday tasks
  3. MFA
    • Supportoed MFA devices
      • Virtual MFA -> software app that runs on phone or device (e.g. google authenticator, twilio)
      • Hardware TOTP token -> Hardware device (key fob, display card)
      • Fido security keys (FIDO certified hardware security keys)

1.3.4. AWS Identity and Access Management

  1. Authentication and Authorization

    Authentication: The process when the user use a conmbination of email and password to verfify the identity. if the system assume the user is allowed to enter grants access Authorization: After your Authentication is the process of give user permission to access different resources and services.

  2. What is IAM??

    Is a services that help you to manage access to your aws account and services in a granular way Global, integrated with Services, shared access without sharing your credentials, support MFA and free to use.

  3. IAM Users

    Represents the service or person that interacts with your account.

  4. IAM User Credentials

    Set of key/value that you give to the users to access through the AWS management console or programattic access to the aws CLI and AWS API

  5. IAM Groups

    Collection of users with same permissions.

    • Groups can have many users
    • Users can belong to many groups
    • Groups Cannot belong to groups
  6. IAM policies

    To manage accessa and provide permisions you create IAM policies and attache them to an IAM identity
